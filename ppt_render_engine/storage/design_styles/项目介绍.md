# 项目介绍 · 设计规范

> **设计原则**：每页必须包含 2 种以上视觉元素（色块、表格、箭头、菱形、流程图等），禁止纯文字堆砌。逻辑关系必须用图形表达。

## 配色

- **主色**: #2E75B6（深邃蓝）
- **强调色**: #4472C4（标准蓝）
- **辅色**: #5B9BD5（浅蓝）、#70AD47（绿色）、#ED7D31（橙色）
- **文字**: #000000 / #404040 / #FFFFFF

形状背景色使用上述色系，文字白色加粗。每个色块用矩形或圆角矩形包裹，背景色填充。

## 模板

- 使用 **标准1** 模板
- 版式选用：标题幻灯片、节标题、标题和内容、两栏内容

## 逻辑图形模式

### 1. 步骤流程（横向）
使用多个圆角矩形或矩形横向排列，用箭头形状（right_arrow）连接：

```
[步骤1] → [步骤2] → [步骤3] → [步骤4]
```

每格配色：#2E75B6（蓝）→ #4472C4（蓝）→ #5B9BD5（浅蓝）→ #70AD47（绿），递进关系用颜色深浅表达。

### 2. 分支判断（菱形）
使用 diamond 形状表示条件判断，搭配 rounded_rectangle 表示结果状态：

```
       [条件 A]
      /         \
 [分支1]     [分支2]
```

diamond 填充 #ED7D31（橙色），结果用 #4472C4（蓝）或 #70AD47（绿）。

### 3. 架构层级（纵向 y）
在容器内纵向堆叠多行矩形，每行代表一层：

- 顶层（宽矩形，#2E75B6）：**应用层** — 标题
- 中层（窄矩形，等宽，#4472C4）：**服务层** — 文字描述
- 底层（窄矩形，等宽，#5B9BD5）：**数据层** — 文字描述

### 4. 对比表格
使用 table，3 列（项目 / 方案A / 方案B），表头加粗居中，列宽均匀。

### 5. 功能矩阵（2×2 / 3×2 色块）
在容器内排列多个圆角矩形卡片，每张有标题和一行描述文字。行内左对齐，卡片间留 200000 EMU 间距。

### 6. 流程图（圆角矩形 + 箭头）
使用 flowterminal（开始/结束）和 flowprocess/椭圆（处理步骤），加上 right_arrow / down_arrow 连接。流程图专用形状：
- flowterminal: 开始/结束，填充 #70AD47
- flowprocess: 步骤，填充 #4472C4
- flowdecision: 判断（等同于 diamond）

## 页面结构

1. **封面** — 标题幻灯片，标题+副标题居中
2. **节过渡页** — 节标题版式，大标题居中+描述文字
3. **内容页（架构图）** — 标题和内容，容器内从上到下排列三层矩形（应用层/服务层/数据层），各配文字
4. **内容页（流程对比）** — 两栏内容，左栏横向步骤流程（箭头串联色块），右栏对比表格
5. **内容页（决策流程）** — 标题和内容，容器内菱形判断 + 左右两路分支矩形
6. **内容页（功能矩阵）** — 标题和内容，容器内 2×3 六宫格彩色卡片
7. **结尾** — 标题幻灯片，结束语
